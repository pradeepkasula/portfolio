GenAI & Agentic AI Work

I am actively experimenting with Generative AI, RAG, and Agentic AI to augment developers and domain experts working on insurance and enterprise applications.

Most of these projects start as internal prototypes or experiments, with a strong focus on reliability, guardrails, and measurable impact on delivery speed.

Example AI Projects

Developer Knowledge Copilot (RAG)

A retrieval‑augmented copilot that allows developers to query internal documentation, API specifications, and code examples using natural language instead of manual search.

  • Ingested design docs, API contracts, and code snippets into a vector store.
  • Built a RAG pipeline to retrieve relevant context and generate grounded answers.
  • Focused on explaining domain‑specific concepts in benefits/insurance systems.
RAG LLM Orchestration Developer Experience

Incident & Log Analysis Assistant

An AI assistant that helps engineers interpret application logs and error traces, suggesting likely root causes and next debugging steps for production issues.

  • Used LLMs with structured prompts to summarize error bursts and patterns.
  • Integrated with existing logging/monitoring tools to fetch relevant context.
  • Emphasized transparency by always showing raw evidence and reasoning steps.
Agentic AI Observability Reliability Engineering

Requirements Clarification Bot

A conversational assistant that refines vague business requirements into clearer user stories and acceptance criteria, improving communication between product and engineering.

  • Guides stakeholders through structured clarification questions.
  • Generates candidate user stories, edge cases, and test ideas.
  • Designed with explicit safety prompts and review loops.
